    Create `packages/editor-core/src/regeneration-replay.ts`. Export:
    - `previewRegenerationReplay(input: { previousBundle: ProjectBundle; regeneratedBundle: ProjectBundle; patchIds?: string[]; operationIds?: string[]; createdAt?: string })`
    - `applyRegenerationReplay(input: { previousBundle: ProjectBundle; regeneratedBundle: ProjectBundle; replay: RegenerationReplayRecord; createdAt?: string })`
    - `appendRegenerationReplay(bundle, replay, actorId?)`

    Behavior requirements:
    - Replay candidates come from stored `patches` and `canvasOperations` on the previous bundle.
    - A patch can replay only when target `nodeId` exists in regenerated `editGraph`, target kind/tag is compatible, patch value is safe for that op, and base revision is understood.
    - A canvas operation can replay only when target `objectId` exists in regenerated `canvasGraph`, operation value passes the same validation used by `applyCanvasOperationToBundle`, and operation does not create graph cycles or missing component references.
    - Missing targets, stale revisions, incompatible node/object kinds, unsafe values, and operation rejection produce `RegenerationReplayConflict` entries.
    - `previewRegenerationReplay` must not mutate either bundle.
    - `applyRegenerationReplay` applies only when replay status is `applied`; conflict/rejected records must return the regenerated bundle unchanged except for persisted replay diagnostics.
    - Replayed patches/operations must get new ids and `baseRevision` equal to the regenerated bundle revision.
    - `appendRegenerationReplay` should append an activity/audit record when collaborators from 11-02 exist.
    - Add `export * from "./regeneration-replay.js";` to `packages/editor-core/src/index.ts`.

    Add tests for successful patch replay, successful canvas operation replay, missing node conflict, missing object conflict, incompatible kind conflict, unsafe style/value conflict, graph-cycle operation rejection, no mutation during preview, conflict apply no-op, and reload persistence.
  </action>
