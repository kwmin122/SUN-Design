  <action>
    Create `packages/editor-core/src/collaboration.ts`. Export:
    - `addCollaborator(bundle, input)`
    - `setCollaboratorRole(bundle, collaboratorId, role, actorId, createdAt?)`
    - `createPresenceRecord(bundle, input)`
    - `setFollowSpotlight(bundle, input)`
    - `createAnnotationThread(bundle, input)`
    - `replyToAnnotationThread(bundle, input)`
    - `resolveAnnotationThread(bundle, threadId, actorId, createdAt?)`
    - `requestReview(bundle, input)`
    - `setReviewState(bundle, input)`
    - `approveReview(bundle, input)`
    - `appendActivityEvent(bundle, input)`
    - `appendAuditLogEntry(bundle, input)`
    - `createRoleBasedShareLink(bundle, input)`

    Behavior requirements:
    - Helper functions return a new `ProjectBundle`; they do not mutate inputs.
    - All created ids use `stableHash`.
    - Unknown collaborator ids throw deterministic errors.
    - Annotation targets must exist: `objectId`, `nodeId`, `slideId`, or `artifactId` references are validated before insertion.
    - `implementationNote` annotation threads must require a target object/node/artifact and cannot be project-only.
    - Presence can reference a selected object/node but cannot reference missing targets.
    - Follow/spotlight cannot target the same collaborator as follower and target.
    - Review records must reference existing target objects when `targetObjectIds` is non-empty.
    - Approvals must reference an existing review, matching reviewer/approver when specified, and current `sourceRevision`.
    - Every role change, review transition, approval, annotation creation, and resolved thread must append both an activity event and an audit log entry.
    - `createRoleBasedShareLink` may reuse the existing `ShareLinkSchema`, but it must preserve role intent in audit/activity records and must not create external URLs.

    Add `export * from "./collaboration.js";` to `packages/editor-core/src/index.ts`.

    Add tests for collaborator creation, role changes, presence, follow/spotlight, annotation threads, implementation notes, replies, resolving, review request, approve/reject, activity/audit creation, missing target rejection, stale revision rejection, self-follow rejection, and reload through `ProjectBundleSchema`.
  </action>
