    Behavior requirements:
    - Helper functions return a new `ProjectBundle`; they do not mutate inputs.
    - All created ids use `stableHash`.
    - Unknown collaborator ids throw deterministic errors.
    - `createSharedDocumentAccess` must prove shared document semantics locally: two distinct collaborators can be participants for the same `projectId`/access id and open independent sessions against the same `ProjectBundle`.
    - `openDocumentSession` must resolve role from explicit role override, collaborator role, or default role, and append a `documentOpened` activity event.
    - `canCollaboratorPerformAction` must implement the role policy: owner can manage access/share/edit/comment/review/approve, editor can edit/comment/review, commenter can comment/annotate/review, viewer can view only.
    - Denied permission decisions must be persisted and audited with `permissionDenied`; denied edit/comment/review/share helpers must not mutate the document.
    - Annotation targets must exist: `objectId`, `nodeId`, `slideId`, or `artifactId` references are validated before insertion.
    - `createAnnotationThread`, `replyToAnnotationThread`, `requestReview`, `approveReview`, `setCollaboratorRole`, and `createRoleBasedShareLink` must call permission checks before mutating state.
    - `implementationNote` annotation threads must require a target object/node/artifact and cannot be project-only.
    - Presence can reference a selected object/node but cannot reference missing targets.
    - Follow/spotlight cannot target the same collaborator as follower and target.
    - Review records must reference existing target objects when `targetObjectIds` is non-empty.
    - Approvals must reference an existing review, matching reviewer/approver when specified, and current `sourceRevision`.
    - Every role change, review transition, approval, annotation creation, and resolved thread must append both an activity event and an audit log entry.
    - `createRoleBasedShareLink` may reuse the existing `ShareLinkSchema`, but it must preserve role intent in audit/activity records and must not create external URLs.

    Add `export * from "./collaboration.js";` to `packages/editor-core/src/index.ts`.

    Add tests for collaborator creation, shared document access creation, two collaborators opening independent sessions on the same document, role-based permission allow/deny decisions, viewer edit denial with no mutation, commenter comment allowed but edit denied, editor edit allowed, owner access management, role changes, presence, follow/spotlight, annotation threads, implementation notes, replies, resolving, review request, approve/reject, activity/audit creation, missing target rejection, stale revision rejection, self-follow rejection, and reload through `ProjectBundleSchema`.
