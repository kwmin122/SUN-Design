    - Add `CodeRoundtripPackageSchema` with fields `id`, `runtime: AgentRuntimeSchema`, `sourceRevision`, `artifactIds`, `instructionPath`, `manifestJson`, `createdAt`.

    - `createExportArtifactRecord(bundle: ProjectBundle, input: { jobId: string; kind: ExportKind; viewport: PreviewDevice; filename: string; mimeType: string; bytes: number; sha256: string; filePath: string; diagnostics?: string[]; createdAt?: string }): ExportArtifact`
    - `createExportVerification(input: { artifactId: string; kind: "visual-diff" | "signature" | "manifest" | "roundtrip"; status: "passed" | "failed" | "degraded"; expectedHash?: string; actualHash?: string; diagnostics?: string[]; createdAt?: string }): ExportVerification`
    - `createPublishPreview(bundle: ProjectBundle, input: { artifactIds: string[]; viewports: PreviewDevice[]; createdAt?: string }): PublishPreview`
    - `appendExportArtifact(bundle: ProjectBundle, artifact: ExportArtifact, verification?: ExportVerification): ProjectBundle`
    - `appendPublishPreview(bundle: ProjectBundle, preview: PublishPreview): ProjectBundle`

    Required behavior:
    - `createExportArtifactRecord` must reject artifacts whose `sourceRevision !== bundle.baseRevision`.
    - `createPublishPreview` must create `url: "kdesign://publish/{projectId}/{publishId}"` and must not use `https://` unless a real hosted backend exists.
    - `appendExportArtifact` must require that the referenced `ExportJob` exists in `bundle.exportJobs`.
    - `appendPublishPreview` must require all `artifactIds` exist in `bundle.exportArtifacts`.

    Create `packages/editor-core/src/code-roundtrip.ts`. Export:
    - `createCodeRoundtripPackage(bundle: ProjectBundle, input: { runtime: AgentRuntime; artifactIds: string[]; createdAt?: string }): CodeRoundtripPackage`
    - `validateCodeRoundtripImport(bundle: ProjectBundle, input: { packageId: string; runtime: AgentRuntime; sourceRevision: string; patches?: EditPatch[]; operations?: CanvasOperation[]; createdAt?: string }): CodeRoundtripImport`
    - `appendCodeRoundtripPackage(bundle: ProjectBundle, pkg: CodeRoundtripPackage): ProjectBundle`
    - `appendCodeRoundtripImport(bundle: ProjectBundle, result: CodeRoundtripImport): ProjectBundle`

    Required behavior:
    - Package `manifestJson` must contain serialized `ProjectBundle`, `baseRevision`, `canvasGraph`, `editGraph`, `assets`, `designSystem`, `sourceRecords`, `exportArtifacts`, and `instructionsPath`.
    - Roundtrip import must reject mismatched `runtime`, mismatched `sourceRevision`, missing object ids, missing node ids, unsupported patch ops, unsupported canvas operations, and unsafe patch values.
    - Allowed patch ops are `setText`, `setStyle`, `replaceAsset`, `setAttr`, `move`, `resize`, `align`, `reorder`, `setVisibility`.
    - Allowed canvas operations are `setObjectName`, `setObjectVisibility`, `setObjectLock`, `reorderObject`, `groupObjects`, `ungroupObjects`, and `setLayoutConstraints`.
    - `status: "validated"` is allowed only when all references and operations pass.
    - Conflicts must be stored as diagnostics; do not apply patches in this helper.

    Add exports to `packages/editor-core/src/index.ts` for both modules. Add tests for artifact sourceRevision mismatch, publish preview URL shape, missing export job, missing artifact id, package manifest contents, runtime mismatch, sourceRevision mismatch, unsafe patch rejection, and unsupported operation rejection.

    Create `apps/export-worker/src/pptx.ts`. Export:
    - `createRasterizedPptx(bundle: ProjectBundle, input: { deckId?: string; pngArtifactPaths: string[]; createdAt?: string }): Uint8Array`
    - `createEditableSubsetPptx(bundle: ProjectBundle, input: { deckId?: string; createdAt?: string }): { data: Uint8Array; diagnostics: string[] }`
    Required behavior: valid PPTX ZIP parts `[Content_Types].xml`, `_rels/.rels`, `ppt/presentation.xml`, `ppt/slides/slide1.xml`, and relationship files. Unsupported nodes must add diagnostics starting with `unsupported-pptx-node:`.

    Create `apps/export-worker/src/animation.ts`. Export:
    - `exportAnimationGif(frames: Uint8Array[], input: { width: number; height: number; delayMs: number }): Uint8Array`
    - `exportAnimationMp4(framePaths: string[], input: { outputPath: string; fps: number }): Promise<void>`
    Required behavior: GIF must produce bytes starting with `GIF89a`; MP4 must call the bundled ffmpeg path from `@ffmpeg-installer/ffmpeg` and write a non-empty `.mp4` file, or throw `mp4-export-failed` with stderr included.

    Create `apps/export-worker/src/export-worker.ts`. Export:
    - `materializeStoredStateExport(input: { bundle: ProjectBundle; kind: ExportKind; viewport: PreviewDevice; outputDir: string; mode?: "rasterizedPptx" | "editableSubsetPptx"; createdAt?: string }): Promise<{ bundle: ProjectBundle; artifact: ExportArtifact; verifications: ExportVerification[] }>`
    - `materializeStaticPublishPreview(input: { bundle: ProjectBundle; outputDir: string; createdAt?: string }): Promise<{ bundle: ProjectBundle; preview: PublishPreview; artifacts: ExportArtifact[] }>`
    - `materializeCodeRoundtripPackage(input: { bundle: ProjectBundle; runtime: AgentRuntime; outputDir: string; createdAt?: string }): Promise<{ bundle: ProjectBundle; package: CodeRoundtripPackage; artifact: ExportArtifact }>`

    Required materialization behavior:
    - HTML writes standalone HTML from `createStandaloneHtml(bundle)`.
    - ZIP writes `index.html`, `manifest.json`, `project-bundle.json`, `source-notes.md` when present, and `design-context.md` when present.
    - PNG and PDF use Playwright to render standalone HTML from stored state in deterministic desktop/tablet/mobile viewports and write real `.png`/`.pdf` files.
    - PPTX supports both `rasterizedPptx` and `editableSubsetPptx`; rasterized PPTX uses PNG artifact paths, editable subset uses stored canvas/edit graph data.
    - GIF and MP4 are allowed only when `bundle.creation.mode === "prototype"` or `bundle.creation.mode === "slideDeck"` equivalent stored mode exists; otherwise create a failed verification diagnostic `animation-template-required`.
    - Every artifact must call editor-core `createExportArtifactRecord` and `createExportVerification`.
    - Tests must assert file signatures: PNG starts with `89 50 4E 47`, PDF starts with `%PDF`, ZIP/PPTX starts with `50 4B`, GIF starts with `GIF89a`, MP4 file has non-zero bytes and `.mp4` extension.
    - Tests must assert exported HTML/ZIP/PDF/PPTX content does not contain `data-preview-bridge`, `selection-overlay`, or `live iframe DOM`.

    Create `apps/export-worker/src/index.ts` re-exporting worker, zip, pptx, and animation modules.

    Add `.kdesign/exports/` and `.tmp-exports/` to `.gitignore`.
