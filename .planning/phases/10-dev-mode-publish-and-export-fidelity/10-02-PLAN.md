  <action>
    Create `apps/export-worker/src/zip.ts`. Export:
    - `writeDeterministicZip(files: Array<{ path: string; data: Uint8Array | string }>): Uint8Array`
    - `sha256Hex(data: Uint8Array | string): string`
    Required behavior: stable file ordering by `path`, UTF-8 encoding for string inputs, deterministic timestamps, and no absolute paths.

    Create `apps/export-worker/src/pptx.ts`. Export:
    - `createRasterizedPptx(bundle: ProjectBundle, input: { deckId?: string; pngArtifactPaths: string[]; createdAt?: string }): Uint8Array`
    - `createEditableSubsetPptx(bundle: ProjectBundle, input: { deckId?: string; createdAt?: string }): { data: Uint8Array; diagnostics: string[] }`
    Required behavior: valid PPTX ZIP parts `[Content_Types].xml`, `_rels/.rels`, `ppt/presentation.xml`, `ppt/slides/slide1.xml`, and relationship files. Editable subset supports text nodes, image nodes with existing asset records, and `vectorLike`/`button`/`frame` nodes as simple rectangle shapes. Unsupported nodes must add diagnostics starting with `unsupported-pptx-node:`.

    Create `apps/export-worker/src/animation.ts`. Export:
    - `exportAnimationGif(frames: Uint8Array[], input: { width: number; height: number; delayMs: number }): Uint8Array`
    - `exportAnimationMp4(framePaths: string[], input: { outputPath: string; fps: number }): Promise<void>`
    Required behavior: GIF must produce bytes starting with `GIF89a`; MP4 must call the bundled ffmpeg path from `@ffmpeg-installer/ffmpeg` and write a non-empty `.mp4` file, or throw `mp4-export-failed` with stderr included.

    Create `apps/export-worker/src/export-worker.ts`. Export:
    - `materializeStoredStateExport(input: { bundle: ProjectBundle; kind: ExportKind; viewport: PreviewDevice; outputDir: string; mode?: "rasterizedPptx" | "editableSubsetPptx"; createdAt?: string }): Promise<{ bundle: ProjectBundle; artifact: ExportArtifact; verifications: ExportVerification[] }>`
    - `materializeStaticPublishPreview(input: { bundle: ProjectBundle; outputDir: string; createdAt?: string }): Promise<{ bundle: ProjectBundle; preview: PublishPreview; artifacts: ExportArtifact[] }>`
    - `materializeCodeRoundtripPackage(input: { bundle: ProjectBundle; runtime: AgentRuntime; outputDir: string; createdAt?: string }): Promise<{ bundle: ProjectBundle; package: CodeRoundtripPackage; artifact: ExportArtifact }>`

    Required materialization behavior:
    - HTML writes standalone HTML from `createStandaloneHtml(bundle)`.
    - ZIP writes `index.html`, `manifest.json`, `project-bundle.json`, `source-notes.md` when present, and `design-context.md` when present.
    - PNG and PDF use Playwright to render standalone HTML from stored state in deterministic desktop/tablet/mobile viewports and write real `.png`/`.pdf` files.
    - PPTX supports both `rasterizedPptx` and `editableSubsetPptx`.
    - GIF and MP4 are allowed only for prototype or slide-deck artifacts; otherwise produce failed verification diagnostic `animation-template-required`.
    - Every artifact must call editor-core `createExportArtifactRecord` and `createExportVerification`.
    - Tests must assert file signatures: PNG starts with `89 50 4E 47`, PDF starts with `%PDF`, ZIP/PPTX starts with `50 4B`, GIF starts with `GIF89a`, MP4 file has non-zero bytes and `.mp4` extension.
    - Tests must assert exported HTML/ZIP/PDF/PPTX content does not contain `data-preview-bridge`, `selection-overlay`, or `live iframe DOM`.
  </action>
