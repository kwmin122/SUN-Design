  <action>
    Create `packages/editor-core/src/dev-mode.ts`. Export these functions:

    - `createDevModeInspectReport(bundle: ProjectBundle, input: { objectId: string; renderedRect?: { x: number; y: number; width: number; height: number }; createdAt?: string }): DevModeInspectReport`
    - `createDevCodeSnippet(bundle: ProjectBundle, input: { objectId: string; kind: DevCodeSnippetKind; createdAt?: string }): DevCodeSnippet`
    - `markReadyForDev(bundle: ProjectBundle, input: { objectId: string; status?: "ready" | "blocked"; label: string; reviewer?: string; notes?: string; createdAt?: string }): ProjectBundle`
    - `markReadyForDevChanged(bundle: ProjectBundle, objectId: string, createdAt?: string): ProjectBundle`
    - `createVersionDiffRecord(bundle: ProjectBundle, input: { fromRevision: string; toRevision?: string; objectIds?: string[]; createdAt?: string }): VersionDiffRecord`
    - `createAssetDownloadRecord(bundle: ProjectBundle, input: { assetId: string; createdAt?: string }): AssetDownloadRecord`
    - `appendDevModeReport(bundle: ProjectBundle, report: DevModeInspectReport): ProjectBundle`
    - `appendDevCodeSnippet(bundle: ProjectBundle, snippet: DevCodeSnippet): ProjectBundle`
    - `appendVersionDiffRecord(bundle: ProjectBundle, diff: VersionDiffRecord): ProjectBundle`
    - `appendAssetDownloadRecord(bundle: ProjectBundle, record: AssetDownloadRecord): ProjectBundle`

    Required behavior:
    - All functions must call `ensureCanvasGraph` before reading objects.
    - Unknown `objectId` must throw `Unknown Dev Mode object: ${objectId}`.
    - `createDevModeInspectReport` must derive `nodeId`, `bounds`, `spacing`, `layout`, CSS properties, token references, component metadata, prototype metadata, and asset ids from stored `ProjectBundle` fields.
    - If `renderedRect` is absent, use deterministic fallback bounds from `CanvasObject.constraints` and emit an accessibility note with `code: "missing_rendered_rect"`.
    - Token references must use `bundle.designSystem.tokens` and include `cssVariable` and `tailwindClass` from token code mappings when present.
    - Component metadata must include `componentId`, `componentName`, `variantId`, `state`, and `overrideCount` for component instances.
    - Prototype metadata must include interaction ids where the selected object is source or target.
    - `createDevCodeSnippet` must output copyable code:
      - `css`: CSS declaration block from inspect report css properties.
      - `tailwind`: space-separated Tailwind classes from matching token mappings and layout hints.
      - `reactProps`: JSON object string with object id, component id, variant id, overrides, and asset ids.
      - `tokenReference`: newline-separated token names with CSS variable or Tailwind mapping.
    - `markReadyForDev` must add or replace a marker for the same `objectId` at the current `bundle.baseRevision`.
    - `markReadyForDevChanged` must set an existing marker to `status: "changed"` when the bundle base revision no longer matches the marker source revision.
    - `createVersionDiffRecord` must compare current object names, node text previews, constraints, asset ids, component instance state, and prototype interaction references against matching stored `versions[].html` or current revision metadata. If no matching historical HTML exists, it must still create a record with a `kind: "unknown"` info change explaining `missing-version-source`.
    - `createAssetDownloadRecord` must require an existing `AssetRef`, use an existing `ProjectAssetUrl` when present, and throw `Asset download requires stable project URL: ${assetId}` if no stable URL exists.
    - Add `export * from "./dev-mode.js";` to `packages/editor-core/src/index.ts`.

    Add `packages/editor-core/src/__tests__/dev-mode.test.ts` covering inspect reports, each snippet kind, ready marker reload shape, changed marker, version diff with missing version source, asset download URL requirement, component metadata, prototype metadata, and unknown object/asset rejection.
  </action>
