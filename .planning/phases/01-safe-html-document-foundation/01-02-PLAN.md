<objective>
Build the Phase 01 browser path on top of the editor-core foundation: app-owned preview runtime, sandboxed iframe rendering, strict postMessage validation, local-first reloadable state, and a compact diagnostics surface. The app proves that a fixture/imported artifact can be sanitized, normalized, persisted, reloaded, previewed in a sandbox, and monitored without treating the live iframe DOM as durable state.

Requirements fulfilled: CORE-02, CORE-03, CORE-04, PREV-01, PREV-03, SEC-01, SEC-02, SEC-03
Phase goal contribution: This turns the core source-of-truth model into an end-to-end Phase 01 product slice while preserving all deferred boundaries: no real AI generation, no direct visual editing, no Tweaks panel, no responsive preview modes, and no export.
Complexity: L
</objective>

    - `type PreviewMessageValidationResult = { ok: true; message: PreviewMessage } | { ok: false; failure: BridgeValidationFailure }`
    - `validatePreviewMessage(event: MessageEvent, expected: { source: Window | null; nonce: string }): PreviewMessageValidationResult`

    `validatePreviewMessage` must check in this order:
    1. `event.source === expected.source`; otherwise return `code: "invalid_source"`.
    2. `event.data` is an object; otherwise return `code: "schema_invalid"`.
    3. `event.data.type` is one of `"preview.ready"`, `"preview.runtime-error"`, `"preview.console"`; otherwise return `code: "unknown_type"`.
    4. `event.data.nonce` exists; otherwise return `code: "missing_nonce"`.
    5. `event.data.nonce === expected.nonce`; otherwise return `code: "invalid_nonce"`.
    6. `PreviewMessageSchema.safeParse(event.data)` succeeds; otherwise return `code: "schema_invalid"`.
