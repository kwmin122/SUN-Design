<objective>
Create the greenfield TypeScript workspace and the `packages/editor-core` source-of-truth layer for one imported or fixture HTML artifact. This plan builds real schemas, deterministic fixtures, static-first sanitization, stable `data-cdx-id` normalization, an initial `EditGraph`, and serializable project persistence helpers.

Requirements fulfilled: CORE-02, CORE-03, CORE-04, SEC-01, SEC-03
Phase goal contribution: This establishes the durable `ProjectBundle + EditGraph + patch log + assets` foundation that later preview, editing, and export work must consume. It deliberately does not implement AI generation, direct visual editing, Tweaks, responsive preview modes, or export.
Complexity: L
</objective>

    - `sanitizeHtml(input: string): { html: string; report: SanitizerReport; assets: AssetRef[] }`
    - `isDangerousUrl(value: string): boolean`
    - `sanitizeInlineStyle(style: string, path: string, changes: SanitizerChange[]): string`

    Sanitizer policy:
    - Remove elements with tag names `script`, `iframe`, `object`, `embed`, `form`, `input`, `textarea`, `select`, `base`, `meta`, and `link` unless the link is a stylesheet that is converted into a blocked asset record. Phase 01 is static-first and should not keep generated scripts or form controls, but plain `button` elements may remain after unsafe attributes are stripped so the normalizer can classify button candidates.
    - Remove any attribute whose lowercase name starts with `on`.
    - Remove `srcdoc`.
    - Remove `target` attributes from `a` tags.
    - For `href`, `src`, `poster`, and `xlink:href`, remove the attribute when `isDangerousUrl` returns true for schemes `javascript:`, `data:text/html`, `vbscript:`, or protocol-relative `//` URLs.
    - For `style`, parse declarations with PostCSS and remove declarations whose value contains `url(`, `expression(`, `@import`, or `javascript:`.
    - Record every removed element, removed attribute, blocked URL, and rewritten style in `SanitizerReport.changes`.
    - Collect image assets into `AssetRef` records with deterministic ids derived from the source URL, `kind: "image"`, `status: "unknown"`, and `sourceUrl`.

    Create `packages/editor-core/src/normalize.ts` using `parse5`. Export:
    - `normalizeHtml(input: { html: string; title: string; id: string; filename?: string }): ProjectBundle`
    - `buildEditGraph(normalizedHtml: string): EditGraph`
    - `classifyElement(tagName: string, attrs: Record<string, string>, textPreview: string): EditNodeKind`

    `normalizeHtml` must:
    - Call `sanitizeHtml` first.
    - Inject stable `data-cdx-id` attributes into editable candidates while preserving existing `data-cdx-id` if it already starts with `cdx_`.
    - Add `data-cdx-role` for every candidate using the `EditNode.kind`.
    - Create a `ProjectBundle` with `source.kind: filename ? "imported" : "fixture"`, empty `patches`, the sanitizer report, the collected assets, and `html.raw`, `html.sanitized`, `html.normalized`.

    Candidate classification:
    - `main`, `section`, `article`, `header`, `footer`, `nav`, `div` become `frame` or `block`; use `frame` for structural tags and `block` for `div`.
    - `h1`, `h2`, `h3`, `h4`, `h5`, `h6`, `p`, `span`, `li`, `strong`, `em` become `text` when non-empty text is present.
    - `img`, `picture`, `source` become `image`.
    - `a` becomes `button` when it has text content or class containing `cta` or `button`; `button` becomes `button` after sanitizer strips inline event handlers.
    - Removed form-like controls do not appear after sanitization. If a future retained control is encountered, classify as `control`.
    - Unknown retained elements become `unknown`; decorative empty elements become `decorative`.

    `buildEditGraph` must produce `version: 1`, `rootNodeIds`, and a `nodes` record keyed by `data-cdx-id`. Each `EditNode` must include `id`, `kind`, `tagName`, `parentId` when available, `domPath`, `fingerprint`, `editableProps`, and optional `textPreview` or `assetId`. Use editable props:
